# 考研搭子 · 异地访问指南

## 数据安全说明

对象访问网站时，**所有数据 100% 存储在你的 Windows 电脑本地 SQLite 数据库中**
（`D:\study-buddy data\studybuddy.db`），Cloudflare / Ngrok 仅提供加密传输通道，
**不存储、不缓存、不分析**任何用户数据。账号密码通过 HTTPS 加密传输，第三方无法窃取。

---

## 方案一：Cloudflare Tunnel（首选，稳定免费）

**优点**：永久免费、无流量限制、HTTPS 加密、全球 CDN 加速、不掉线、无需注册账号

**对象操作**：打开浏览器输入你发的网址即可，无需安装任何软件

---

### 第 1 步：双击运行（1 秒完成）

双击 `start-tunnel.bat`

脚本会自动：
1. 启动本地 Node.js 服务器
2. 启动 Cloudflare 隧道
3. 在窗口中显示公网域名

### 第 2 步：复制域名发给对象

隧道启动后，窗口中会显示类似这样的信息：

```
+--------------------------------------------------------------------------------------------+
|  Your quick Tunnel has been created! Visit it at:                                           |
|  https://xxx-xxx-xxx.trycloudflare.com                                                     |
+--------------------------------------------------------------------------------------------+
```

把 `https://xxx-xxx-xxx.trycloudflare.com` 复制发给对象即可。

### 第 3 步：对象访问

对象在浏览器（Safari / Chrome / Edge）输入你发的网址，直接就能访问网站。
注册账号、登录、组队、打卡，和本地使用完全一样。

---

### 注意事项

- 每次双击 `start-tunnel.bat` 会生成**新的域名**，需要重新发一次给对象
- 关闭隧道窗口即可停止异地访问（本地服务器会继续在后台运行）
- 如果遇到连接问题，先运行 `firewall-allow.bat` 放行防火墙

---

## 方案二：Ngrok 备用方案

**适用场景**：Cloudflare 隧道不可用时的应急方案

**缺点**：免费版每 2 小时自动断开、需注册账号

### 一次性配置（仅需 1 次，3 分钟）

1. 打开 https://dashboard.ngrok.com/signup ，用 GitHub/Google 账号注册
2. 登录后，左侧菜单 → `Your Authtoken`，复制 token
3. 打开终端，运行一次：
   ```
   ngrok config add-authtoken 你的token
   ```

### 日常使用

双击 `start-ngrok.bat`

窗口中 `Forwarding` 行显示的 `https://xxx.ngrok-free.app` 即为对象访问地址。

### 2 小时断开后

关闭窗口，重新双击 `start-ngrok.bat`，把新地址发给对象。

## 脚本文件说明

| 文件 | 用途 |
|------|------|
| `start-tunnel.bat` | 一键启动（服务器 + Cloudflare 隧道） |
| `start-ngrok.bat` | 一键启动（服务器 + Ngrok 备用隧道） |
| `firewall-allow.bat` | 防火墙放行（首次使用或连接失败时运行一次） |
| `check-tunnel.bat` | 诊断脚本，检查服务器和隧道状态 |

## 故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 对象打不开网址 | 隧道没启动 | 检查隧道窗口是否在运行 |
| 域名能打开但加载不出 | 连接超时 | 等待 30 秒重试（首次建立需要时间） |
| 提示"不安全" | 用了 HTTP | 确保发给对象的是 `https://` 开头 |
| 访问后页面空白 | 防火墙拦截 | 运行 `firewall-allow.bat` |
| 数据不同步 | 服务器只监听 localhost | 已配置 `0.0.0.0`，无需修改 |
| 隧道突然断开 | 网络波动 | Cloudflare 会自动重连，等待 10 秒 |

## 电脑休眠策略（长期使用建议）

为防止电脑长时间不用自动休眠导致网站不可访问：

1. 打开 `控制面板` → `电源选项`
2. 点击当前计划的 `更改计划设置`
3. `使计算机进入睡眠状态` → 改为 `从不`
4. `关闭显示器` → 可设为 30 分钟（不影响服务器运行）
5. 点击 `保存修改`

## 快速操作总结

```
想让对象访问：
  双击 start-tunnel.bat → 复制 https://xxx.trycloudflare.com → 发给对象

对象说打不开：
  双击 check-tunnel.bat → 查看哪里有问题
  双击 firewall-allow.bat → 放行防火墙

换 Ngrok 试试：
  双击 start-ngrok.bat → 复制 https://xxx.ngrok-free.app → 发给对象

不想让别人访问了：
  关闭隧道窗口即可
```
